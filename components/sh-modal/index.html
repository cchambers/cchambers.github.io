modal

Usage

      
        <sh-modal header="Modal Header" unique-id="string">
          <p>
            Lorem ipsum doler.
          </p>
        </sh-modal>

Modals can be added anywhere within content. It will automatically remove itself from the flow of the document and be re-attached to a container at the bottom of the DOM.

Modals can be triggered by modal-trigger='uniqueId' from any element (like a button) or directly linked to from an anchor tag: href='#uniqueId'

If a user lands on a page with a modal targeted via hash it will open automatically: reload with hash

Reference

Attributes

Attribute Type Description
unique-id String
  • Unique ID.
  • This is a required attribute.
variant String
  • Change the way the modal is rendered.
  • Options: primary (default), secondary, tertiary
header String Content for the modal header.
footer String Content for modal footer.
content-url String URL endpoint for ajaxing content into the modal.
content-selector String Selector to target specific content in the returned ajax call.
overlay String
  • Define the main color for the overlay.
  • Options are all base color variables (accent-ordinal, lowlight-ordinal, highlight-ordinal) and 'none'.

Slots

Slot Description
header HTML content for the header of the modal.
content HTML content for the main area of the modal.
footer HTML content for the footer of the modal.

Public Methods

Function Params Description
open
  • none
Trigger the modal. This will close other open modals.
close
  • clearHash
  • This will close all open modals.
  • clearHash is a boolean that defaults to true.

Events

Event Payload Type Description
modal-opening String
  • Fires before the modal opens.
  • Payload is the ID of the modal that will open.
modal-opened String
  • Fires immediately after the modal opens.
  • Payload is the ID of the modal that has opened.
